\subsection{What is Python}
\label{sec:org8928078}
Python is an advanced \textbf{scripting language} that is being used successfully to \textbf{glue together} large software components. It spans \textbf{multiple platforms}, middleware products, and application domains. Python is an \textbf{object-oriented language} with \textbf{high-level data structures}, \textbf{dynamic typing}, and \textbf{dynamic binding}. Python has been around since 1991, and has a very active user community. For more information, see the Python website \url{http://www.python.org}.\\

\subsection{兩種執行模式}
\label{sec:orgd67482a}
\begin{enumerate}
\item Interactive mode
\label{sec:org4a4a15f}
\begin{enumerate}
\item 啟動方式
\label{sec:orgef298c3}
\lstset{breaklines=true,language=shell,label= ,caption= ,captionpos=b,numbers=none}
\begin{lstlisting}
python -i filename.py
\end{lstlisting}
\item 優點\footnote{\href{https://stackabuse.com/python-programming-in-interactive-vs-script-mode/}{Python Programming in Interactive vs Script Mode}\\}
\label{sec:orgbd2b595}
\begin{itemize}
\item Helpful when your script is extremely short and you want immediate results.\\
\item Faster as you only have to type a command and then press the enter key to get the results.\\
\item Good for beginners who need to understand Python basics.\\
\end{itemize}
\item 缺點
\label{sec:org765b102}
\begin{itemize}
\item Editing the code in interactive mode is hard as you have to move back to the previous commands or else you have to rewrite the whole command again.\\
\item It's very tedious to run long pieces of code.\\
\end{itemize}
\end{enumerate}
\item Script Mode
\label{sec:orgc734433}
\begin{enumerate}
\item 啟動方式
\label{sec:orgd2b4aab}
\lstset{breaklines=true,language=bash,label= ,caption= ,captionpos=b,numbers=none}
\begin{lstlisting}
python filename.py
\end{lstlisting}
\item 優點:
\label{sec:orgd340b7a}
\begin{itemize}
\item It is easy to run large pieces of code.\\
\item Editing your script is easier in script mode.\\
\item Good for both beginners and experts.\\
\end{itemize}
\item 缺點:
\label{sec:orgf5df1d8}
\begin{itemize}
\item Can be tedious when you need to run only a single or a few lines of cod.\\
\item You must create and save a file before executing your code.\\
\end{itemize}
\end{enumerate}
\end{enumerate}

\subsection{for / while + else}
\label{sec:orgdd21f76}
Python 一個十分有趣的 loop 語法是它能配合 else 來用，語法如下：\\
\lstset{breaklines=true,language=Python,label= ,caption= ,captionpos=b,firstnumber=1,numbers=left}
\begin{lstlisting}
for n in ...:
    if condition is true:
        ....
        break
else:
    print("all condition is false")
\end{lstlisting}
如果上述的 for 迴圈從頭到尾都沒有去跑過 break(也就是說 condition 都不成立，for 正常結束)，那麼 else 裡的程式碼就會被執行。那麼，何時會用到這個有趣的語法呢？\\

以質數判斷為例：原本的寫法為:\\
\lstset{breaklines=true,language=Python,label= ,caption= ,captionpos=b,firstnumber=1,numbers=left}
\begin{lstlisting}
n = 22
isPrime = True
for x in range(2, n//2 + 1):
    if n % x == 0:
        isPrime = False
        print("Composite Number!")
        break
if isPrime:
    print("Prime number")
\end{lstlisting}

\begin{verbatim}
Composite Number!
\end{verbatim}


套用 else 後則可改為\\
\lstset{breaklines=true,language=Python,label= ,caption= ,captionpos=b,firstnumber=1,numbers=left}
\begin{lstlisting}
n = 23
for x in range(2, n//2 + 1):
    if n % x == 0:
        print("Composite Number!")
        break
else:
    print("Prime number")

\end{lstlisting}

\begin{verbatim}
Prime number
\end{verbatim}

\subsection{引數(argument)的傳遞}
\label{sec:org66ae7df}
\begin{itemize}
\item 每一個函式都是獨立的，所以函式只知道自己程式區域內的變數，不認識函式外的變數。\\
\item 變數因其所宣告位置，可區分為全域變數(Global variable)、區域變數(Local variable)。\\
\begin{itemize}
\item 全域變數：宣告在任何函式外，所有函式皆可存取。\\
\item 區域變數：宣告在函式內，僅函式內可存取。c\\
\end{itemize}
\item 全域變數與區域變數範例\\
\end{itemize}
\lstset{breaklines=true,language=Python,label= ,caption= ,captionpos=b,firstnumber=1,numbers=left}
\begin{lstlisting}
  def printGlobal(x, y):
      print(a+b)

  def printLocal(a, b):
      print(a+b)

  a, b = 5, 10
  printGlobal(10, 10)
  printLocal(10,10)
\end{lstlisting}

\begin{verbatim}
15
20
\end{verbatim}

\begin{enumerate}
\item 傳值與傳址
\label{sec:orga4e3b24}
\begin{itemize}
\item 引數(argument)的傳遞分為兩種方式，由 Python 自動判別：\\
\begin{itemize}
\item 傳值呼叫(Call-by-Value)\\
\item 傳址呼叫(Call-by-Reference)\\
\end{itemize}
\end{itemize}
\begin{center}
\begin{tabular}{llll}
引數傳遞 & 傳遞內容 & 參數改變 & 資料型別\\
\hline
傳值呼叫 call-by-value & 引數值 & 否 & bool, int, float, str, tuple, chr\\
傳址呼叫 call-by-reference & 引數記憶體位址 & 是 & list, dict, set\\
\end{tabular}
\end{center}
\lstset{breaklines=true,language=Python,label= ,caption= ,captionpos=b,firstnumber=1,numbers=left}
\begin{lstlisting}
  # str傳值、list傳址
  def ccc(name, score):
      name = 'Amy'
      score.append(99)
      print('--函式內輸出---')
      print('name:', name,'\tscore:',score)

  name = 'Josh'
  score = [77, 88]
  
  ccc(name, score)
  print('--主程式輸出--')
  print('name:', name,'\tscore:',score)
\end{lstlisting}

\begin{verbatim}
--函式內輸出---
name: Amy 	score: [77, 88, 99]
--主程式輸出--

\end{verbatim}
\end{enumerate}

\subsection{遞迴(recursive)}
\label{sec:orgc28e92f}
\begin{enumerate}
\item 語法
\label{sec:org1646289}
\begin{itemize}
\item 遞迴(recursive)：一個函式直接呼叫函式本身\\
\end{itemize}
\lstset{breaklines=true,language=Python,label= ,caption= ,captionpos=b,firstnumber=1,numbers=left}
\begin{lstlisting}
  def fib(n):
      if n == 1 or n == 2:
          return 1
      else:
          return fib(n-1) + fib(n-2)

  print('The %d-th Fibonacci number = %d' %(10, fib(10)))
\end{lstlisting}

\begin{verbatim}
The 10-th Fibonacci number = 55
\end{verbatim}

\item 基本條件
\label{sec:orgee3832a}
\begin{itemize}
\item 遞迴程式設計要滿足兩條件:\\
\begin{enumerate}
\item 遞迴關係式，如\\
\lstset{breaklines=true,language=Python,label= ,caption= ,captionpos=b,firstnumber=1,numbers=left}
\begin{lstlisting}
       fib(n) = fib(n-1) + fib(n-2)
       sum(n) = n + sum(n-1) 
\end{lstlisting}
\item 遞迴終止式，如\\
\lstset{breaklines=true,language=Python,label= ,caption= ,captionpos=b,firstnumber=1,numbers=left}
\begin{lstlisting}
        f(1) = f(2) = 1
        sum(0) = 0
\end{lstlisting}
\end{enumerate}
\item 範例:GCD-while\\
\lstset{breaklines=true,language=Python,label= ,caption= ,captionpos=b,firstnumber=1,numbers=left}
\begin{lstlisting}
    m, n = 20, 50
    while (n>0):
        m, n = n, m % n
    print(m)
\end{lstlisting}

\begin{verbatim}
10
\end{verbatim}
\end{itemize}

\item 範例
\label{sec:org0d33433}
\begin{itemize}
\item 範例:GCD-recursive\\
\lstset{breaklines=true,language=Python,label= ,caption= ,captionpos=b,firstnumber=1,numbers=left}
\begin{lstlisting}
    def gcd(a, b):
        if b == 0:
            return a
        return gcd(b, a % b)
  
    a, b = 10, 50
    print(gcd(a, b))
\end{lstlisting}

\begin{verbatim}
10
\end{verbatim}
\end{itemize}
\end{enumerate}
