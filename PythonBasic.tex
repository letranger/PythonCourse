\label{sec:orgc76632b}
\begin{itemize}
\item Python is an advanced \textbf{scripting language} that is being used successfully to
\textbf{glue together} large software components. It spans \textbf{multiple platforms},
middleware products, and application domains. Python is an \textbf{object-oriented
language} with \textbf{high-level data structures}, \textbf{dynamic typing}, and \textbf{dynamic binding}.
Python has been around since 1991, and has a very active user community. For
more information, see the Python website \url{http://www.python.org}.

\subsection{兩種執行模式}
\label{sec:org60b3e28}
\begin{enumerate}
\item Interactive mode
\label{sec:org15da08c}
\begin{enumerate}
\item 啟動方式
\label{sec:org0f0b09e}
\lstset{breaklines=true,language=shell,label= ,caption= ,captionpos=b,numbers=none}
\begin{lstlisting}
python -i filename.py
\end{lstlisting}
\item 優點\footnote{\href{https://stackabuse.com/python-programming-in-interactive-vs-script-mode/}{Python Programming in Interactive vs Script Mode}}
\label{sec:org4651ee9}
\begin{itemize}
\item Helpful when your script is extremely short and you want immediate results.
\item Faster as you only have to type a command and then press the enter key to get the results.
\item Good for beginners who need to understand Python basics.
\end{itemize}
\item 缺點
\label{sec:orgb53f318}
\begin{itemize}
\item Editing the code in interactive mode is hard as you have to move back to the previous commands or else you have to rewrite the whole command again.
\item It's very tedious to run long pieces of code.
\end{itemize}
\end{enumerate}
\item Script Mode
\label{sec:org4988acb}
\begin{enumerate}
\item 啟動方式
\label{sec:orgf82afc7}
\lstset{breaklines=true,language=bash,label= ,caption= ,captionpos=b,numbers=none}
\begin{lstlisting}
python filename.py
\end{lstlisting}
\item 優點:
\label{sec:orgf248046}
\begin{itemize}
\item It is easy to run large pieces of code.
\item Editing your script is easier in script mode.
\item Good for both beginners and experts.
\end{itemize}
\item 缺點:
\label{sec:org07839e6}
\begin{itemize}
\item Can be tedious when you need to run only a single or a few lines of cod.
\item You must create and save a file before executing your code.
\end{itemize}
\end{enumerate}
\end{enumerate}

\item 語法
\lstset{breaklines=true,language=Python,label= ,caption= ,captionpos=b,numbers=none}
\begin{lstlisting}
  變數名稱 = 指派值
\end{lstlisting}

\item 範例：
\lstset{breaklines=true,language=Python,label= ,caption= ,captionpos=b,firstnumber=1,numbers=left}
\begin{lstlisting}
a = 5
b = 3.14
c = 'TNFSH'
a = b = c = 10
name, number = 'TNFSH', 35
del c
print(name)
\end{lstlisting}
\end{itemize}

\begin{enumerate}
\item 能否改變內容
\label{sec:org238e132}
\begin{itemize}
\item Using a tuple instead of a list can give the programmer and the interpreter a hint that the data should not be changed.
\end{itemize}
\item 可讀性
\label{sec:orgecf35ce}
\begin{itemize}
\item Reading data is simpler when tuples are stored inside a list. For example,
\lstset{breaklines=true,language=Python,label= ,caption= ,captionpos=b,firstnumber=1,numbers=left}

\lstset{breaklines=true,language=Python,label= ,caption= ,captionpos=b,firstnumber=1,numbers=left}
\begin{lstlisting}
[[2,4], [5,7], [3,8], [5,9]]
\end{lstlisting}
\end{itemize}
\item Tuple 與 dictionary 的相似性
\label{sec:orgb279236}
\begin{itemize}
\item Tuples are commonly used as the equivalent of a dictionary without keys to store data. For Example,
\lstset{breaklines=true,language=Python,label= ,caption= ,captionpos=b,firstnumber=1,numbers=left}
\begin{lstlisting}
[('Swordfish', 'Dominic Sena', 2001), ('Snowden', ' Oliver Stone', 2016), ('Taxi Driver', 'Martin Scorsese', 1976)]
\end{lstlisting}
\item Above example contains tuples inside list which has a list of movies.
\item Tuple can also be used as key in dictionary due to their hashable and immutable nature whereas Lists are not used as key in a dictionary because list can't handle \_\_hash\_\_() and have mutable nature.
\end{itemize}

\subsection{函式(function)基本操作}
\label{sec:org9a38325}
\begin{enumerate}
\item 函式(function)的重要性
\label{sec:org8b4cce0}
\begin{itemize}
\item function 為一段程式的集合，可視為一個獨立的區段。
\item function 可重覆使用，因此為結構化程式語言的重要元素。 可將龐大複雜的程式分解為小問題，由多人分別以 function 解決，縮短開發時間。
\item function 可區分為以下三類：
\begin{itemize}
\item Python 內建 function，如: print()、len()、int()、str()\ldots{}。
\item 第三方公司所開發的模組 module(多個 function 組合)。
\item 自定義 function。
\end{itemize}
\end{itemize}
\item 自定義函式
\label{sec:orgd7e501a}
\begin{itemize}
\item 宣告語法
\lstset{breaklines=true,language=Python,label= ,caption= ,captionpos=b,firstnumber=1,numbers=left}
\begin{lstlisting}
    def 函式名稱 ( [參數1, 參數2, ...] ):
        程式區塊 
        [ return 值 ]
\end{lstlisting}
\item 說明 :
\begin{itemize}
\item 參數可省略，亦即呼叫 function 不需傳入任何資料(引數)。
\item 若無回傳值，則無需 return。
\item 若多個回傳值可用逗號隔開。
\end{itemize}
\end{itemize}

\subsection{引數(argument)的傳遞}
\label{sec:org3b22e8a}
\begin{itemize}
\item 每一個函式都是獨立的，所以函式只知道自己程式區域內的變數，不認識函式外的變數。
\item 變數因其所宣告位置，可區分為全域變數(Global variable)、區域變數(Local variable)。
\begin{itemize}
\item 全域變數：宣告在任何函式外，所有函式皆可存取。
\item 區域變數：宣告在函式內，僅函式內可存取。c
\end{itemize}
\item 全域變數與區域變數範例
\lstset{breaklines=true,language=Python,label= ,caption= ,captionpos=b,firstnumber=1,numbers=left}
\begin{lstlisting}
    def printGlobal(x, y):
        print(a+b)
  
    def printLocal(a, b):
        print(a+b)
  
    a, b = 5, 10
    printGlobal(10, 10)
    printLocal(10,10)
\end{lstlisting}

\begin{verbatim}
15
20
\end{verbatim}
\end{itemize}

\begin{enumerate}
\item 傳值與傳址
\label{sec:orgc1d5a37}
\begin{itemize}
\item 引數(argument)的傳遞分為兩種方式，由 Python 自動判別：
\begin{itemize}
\item 傳值呼叫(Call-by-Value)
\item 傳址呼叫(Call-by-Reference)
\end{itemize}
\end{itemize}
\begin{center}
\begin{tabular}{llll}
引數傳遞 & 傳遞內容 & 參數改變 & 資料型別\\
\hline
傳值呼叫 call-by-value & 引數值 & 否 & bool, int, float, str, tuple, chr\\
傳址呼叫 call-by-reference & 引數記憶體位址 & 是 & list, dict, set\\
\end{tabular}
\end{center}
\lstset{breaklines=true,language=Python,label= ,caption= ,captionpos=b,firstnumber=1,numbers=left}
\begin{lstlisting}
  # str傳值、list傳址
  def ccc(name, score):
      name = 'Amy'
      score.append(99)
      print('--函式內輸出---')
      print('name:', name,'\tscore:',score)

  name = 'Josh'
  score = [77, 88]
  
  ccc(name, score)
  print('--主程式輸出--')
  print('name:', name,'\tscore:',score)
\end{lstlisting}

\begin{verbatim}

\end{verbatim}
\end{enumerate}

\subsection{遞迴(recursive)}

\item 遞迴(recursive)：一個函式直接呼叫函式本身

  def fib(n):
      if n == 1 or n == 2:

       fib(n) = fib(n-1) + fib(n-2)

\lstset{breaklines=true,language=Python,label= ,caption= ,captionpos=b,firstnumber=1,numbers=left}

        f(1) = f(2) = 1
